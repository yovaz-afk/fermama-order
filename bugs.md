## Найденные баги

### BUG-01 [КРИТИЧНЫЙ] price-preview.html — Кнопки перекрывают заголовок на мобиле
- **Файл:** price-preview.html
- **Воспроизведение:** Открыть на телефоне (≤640px)
- **Ожидаемо:** Кнопки "Поделиться" и "Скачать PDF" не мешают контенту
- **Факт:** `.top-bar { position: fixed; top: 16px; right: 16px }` накладывается на зелёный header
- **Причина:** На мобиле хедер занимает всю ширину, а кнопки поверх
- **Исправление:** Сделать `.top-bar` статичным на мобиле (position: static, кнопки под шапкой)
- **Статус:** ✅ ИСПРАВЛЕНО

---

### BUG-02 [СРЕДНИЙ] price-preview.html — Мелкий шрифт на мобиле
- **Файл:** price-preview.html
- **Воспроизведение:** Открыть прайс на телефоне
- **Ожидаемо:** Таблица товаров читаема, цены крупные
- **Факт:** `font-size: 12px` в таблице, 11px в хитах — слишком мелко
- **Исправление:** Увеличить шрифты в @media (max-width: 640px) до 13-14px
- **Статус:** ✅ ИСПРАВЛЕНО

---

### BUG-03 [КРИТИЧНЫЙ] index.html — Telegram открывает официальный сайт вместо приложения
- **Файл:** index.html, функция `openMessenger()`
- **Воспроизведение:** Добавить товары → ОТПРАВИТЬ → Telegram
- **Ожидаемо:** Открывается Telegram с готовым текстом заявки
- **Факт:** `https://t.me/share/url?url=&text=...` — URL с пустым `url=` ведёт на официальный сайт Telegram, а не в приложение
- **Причина:** `url=` без значения некорректен; на мобиле не открывает приложение
- **Исправление:** Заменить на deep link `tg://msg?text=...` (открывает Telegram на выбор чата)
- **Статус:** ✅ ИСПРАВЛЕНО

### BUG-04 [КРИТИЧНЫЙ] index.html — Кнопка "Очистить" не работает
- **Файл:** index.html, функция `handleReset()`
- **Воспроизведение:** Добавить товары → нажать "Очистить" (кнопка в хедере)
- **Ожидаемо:** Форма очищается
- **Факт:** Ничего не происходит
- **Причина:** `window.confirm()` заблокирован в Telegram WebApp (встроенный браузер) — всегда возвращает `false`, тело функции не выполняется
- **Исправление:** Убрать `window.confirm()`, сразу выполнять сброс + показывать статус "Заявка очищена"
- **Статус:** ✅ ИСПРАВЛЕНО

---

### BUG-05 [МАЛЫЙ] index.html — "Поделиться" fallback тот же битый URL
- **Файл:** index.html, функция `openMessenger('share')`
- **Воспроизведение:** ОТПРАВИТЬ → Поделиться (если Web Share API недоступен)
- **Факт:** Fallback использует тот же `https://t.me/share/url?url=&text=...`
- **Исправление:** Заменить fallback на `tg://msg?text=...`
- **Статус:** ✅ ИСПРАВЛЕНО
