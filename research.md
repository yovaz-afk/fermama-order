# Исследование рынка: приложения приёма заявок для торговых представителей

> **Контекст:** ФерМама — производитель замороженных полуфабрикатов (пельмени, вареники, котлеты, чебуреки, манты и др.). Торговые представители объезжают розничные магазины и принимают заявки через мобильное приложение. Текущий флоу: выбрать себя → ввести клиента → добавить товары с количеством → отправить в Telegram/WhatsApp.

---

## 1. 1С:Мобильная торговля

**Что это:** Мобильное решение от 1С для автоматизации полевых продаж. Используется тысячами торговых представителей по всей России в продуктовом ретейле, FMCG, мясной и молочной промышленности. Интегрируется с 1С:Предприятие.

**Основной флоу:**
1. Авторизация торгпреда (логин/пароль или выбор из списка)
2. Список клиентов с маршрутом на день (геолокация, статус визита)
3. Выбор клиента → карточка клиента (история, долг, условия)
4. Каталог товаров с остатками и ценами для данного клиента
5. Добавление позиций в корзину (количество, скидки)
6. Оформление заявки с датой доставки и комментарием
7. Синхронизация с сервером 1С в онлайн/офлайн режиме

**Сильные стороны:**
- Работает офлайн — нет интернета, приложение не ломается
- Маршрутные листы: торгпред видит план на день по геолокации
- История заказов клиента — видно, что брал в прошлый раз
- Индивидуальные цены и скидки для каждого клиента
- Дебиторская задолженность прямо в карточке клиента
- Фотофиксация выкладки на полке
- Синхронизация с бухгалтерией в реальном времени

**Слабые стороны:**
- Интерфейс выглядит как Windows XP — устаревший дизайн
- Долгая и дорогая настройка (нужен 1С-специалист)
- Тяжёлое приложение, тормозит на бюджетных телефонах
- Обучение сотрудников занимает 2-3 дня
- Дорогая лицензия (от 15 000 руб./месяц на команду)
- Сложно добавить новый товар без программиста
- Интерфейс перегружен — 50+ полей которые торгпреду не нужны

**Что ФерМама может сделать лучше:**
- Максимально простой интерфейс: 3 шага вместо 10
- Мгновенный старт без обучения (интуитивно понятно)
- Отправка через мессенджеры — не нужна интеграция с 1С
- Быстрее запускается, быстрее работает

## 2. МойСклад (moisklad.ru)

**Что это:** Российский облачный сервис для управления торговлей. Имеет B2B-портал (витрину) где оптовые покупатели сами размещают заказы. Активно используется небольшими производителями продуктов питания для работы с розничными клиентами.

**Основной флоу (B2B-портал для клиентов):**
1. Покупатель (магазин) заходит на B2B-витрину производителя
2. Видит каталог с фото, ценами, остатками
3. Добавляет товары в корзину
4. Оформляет заказ (реквизиты уже заполнены)
5. Заказ падает в МойСклад на стороне производителя

**Для торгпреда (мобильное приложение МойСклад):**
1. Выбор контрагента из списка
2. Создание нового заказа покупателя
3. Добавление товаров через поиск или каталог
4. Указание количества, скидки
5. Сохранение/отправка заказа

**Сильные стороны:**
- Красивая B2B-витрина с фото товаров
- Остатки обновляются в реальном времени
- Клиент может сам разместить заказ (без торгпреда)
- История всех заказов и документов в одном месте
- Интеграция с 1С, Wildberries, Ozon
- Мобильное приложение есть
- Доступная цена (от 500 руб./месяц)

**Слабые стороны:**
- B2B-витрина требует времени на настройку и наполнение фото
- Клиентам нужно объяснять, как войти и разместить заказ
- Мобильное приложение для торгпреда — вторичная функция, не очень удобное
- Слабая офлайн-поддержка
- Каталог с фото требует ведения (обновлять фото, описания)
- Не оптимизировано под быстрый визит в магазин "на бегу"
- Торгпред не может мгновенно отправить текст заявки в WhatsApp

**Что ФерМама может сделать лучше:**
- Отправка в один клик в любой мессенджер (Telegram/WhatsApp/Viber)
- Не нужна регистрация клиенту
- Приложение готово к работе без настройки и наполнения каталога

## 3. Pepperi (международный, pepperi.com)

**Что это:** Международная платформа для автоматизации B2B продаж в FMCG и food-сегменте. Используется дистрибьюторами продуктов питания, производителями мяса и молочных продуктов в Европе и США. Включает мобильное приложение для торгпредов и B2B self-service портал.

**Основной флоу:**
1. Торгпред выбирает клиента из маршрута (с геолокацией)
2. Открывает каталог — товары с фото, ценами, доступными объёмами
3. Режим "повторить прошлый заказ" — одна кнопка
4. Корзина с живым расчётом суммы
5. Подпись клиента на экране телефона (электронная)
6. Отправка заказа → уведомление на склад в реальном времени

**Сильные стороны:**
- Каталог с профессиональными фото товаров — как интернет-магазин
- "Повторить заказ" — killer feature для регулярных визитов
- Электронная подпись клиента прямо на экране
- Офлайн-режим полностью рабочий
- Аналитика для менеджера: план/факт, воронка, KPI
- Промо-акции и скидки прямо в каталоге
- Поиск по каталогу с автодополнением

**Слабые стороны:**
- Дорого — от $200/месяц за пользователя
- Требует интеграции с ERP/складской системой
- Сложная настройка (3-6 месяцев внедрения)
- Избыточен для небольшого производителя
- Интерфейс сложный, много функций

**Что ФерМама может сделать лучше:**
- Запустить за 1 день, не за полгода
- Цена — ноль рублей (простой HTML)
- Отправка в мессенджер вместо сложной интеграции
- Идеальная простота для небольшой команды (2 торгпреда)

**Идея из Pepperi — Order Matrix (табличный ввод заказа):**
Ключевая фича для ускорения работы с большим каталогом. Вместо dropdown-строк — таблица: строки = товары, столбец = количество. Торгпред сразу видит весь каталог и вводит числа в ячейки. Для ФерМама с 55 SKU это был бы огромный шаг вперёд. Также из Pepperi: шаблоны повторяющихся заказов — большинство магазинов берут одно и то же каждую неделю.

## 4. Handshake / OrderEase (западные B2B wholesale apps)

**Что это:** Американские приложения для оптовой торговли продуктами питания. Handshake — специализируется на food & beverage дистрибуции, OrderEase — для food producers работающих с ретейлом. Используются небольшими мясными фермами и производителями локального питания.

**Основной флоу:**
1. Торгпред открывает приложение → видит список клиентов
2. Выбирает клиента → история заказов
3. Каталог: товары сгруппированы по категориям с фото
4. Быстрое добавление: нажал → ввёл количество → следующий
5. Мини-корзина всегда видна (количество позиций, сумма)
6. Оформление: дата доставки, адрес, комментарий
7. Email клиенту + уведомление на склад

**Сильные стороны:**
- Очень быстрое добавление товаров (оптимизировано под скорость)
- Каталог с фото — клиент может листать сам, пока торгпред стоит рядом
- Мини-корзина "плавает" поверх каталога — всегда видна сумма
- История: "в прошлый раз брали X пачек" — подсказка
- PDF-подтверждение заказа сразу на email клиента
- Работает офлайн, синхронизируется при появлении интернета

**Слабые стороны:**
- Нет русского языка и локализации для России
- Платный сервис (от $50/месяц)
- Нет интеграции с российскими мессенджерами (Telegram, Viber)
- Требует регистрации клиентов в системе
- Избыточен для команды из 2-3 человек

**Что ФерМама может сделать лучше:**
- Мессенджеры вместо email (в России это актуальнее)
- Нет регистрации — просто написал имя клиента и всё
- Полностью бесплатно, без подписки

## 5. GROTEM/Агент и кустарные решения (российская реальность)

**Что это:** GROTEM/Агент — российское мобильное приложение для Android, ориентированное на полевых торгпредов. Работает офлайн, имеет GPS-трекинг, интеграцию с 1С. Одновременно — реальная картина рынка: 90% производителей среднего размера до сих пор работают через WhatsApp + блокнот + Excel.

**Флоу GROTEM:**
1. Менеджер авторизуется, получает маршрутный лист на день
2. Открывает карточку клиента — долг, история
3. Создаёт заявку, выбирает товары из синхронизированного каталога
4. Оформляет, печатает накладную через Bluetooth-принтер
5. Данные уходят в 1С при появлении интернета

**Флоу кустарного WhatsApp + Excel (реальность большинства):**
1. Торгпред принимает заявку в блокнот или диктовкой в голосовое
2. Вечером вручную набивает сообщение в WhatsApp менеджеру
3. Менеджер перебивает в Excel/1С
4. Тройное дублирование данных — норма

**Сильные стороны GROTEM:**
- Реально работает офлайн
- GPS-трекинг торгпреда для контроля
- Печать накладных на точке
- Интеграция с 1С

**Слабые стороны GROTEM и кустарных решений:**
- Устаревший UX, нет каталога с фото
- Сложное и дорогое внедрение (50-150 тыс. руб.)
- Синхронизация с 1С — хроническая боль, сбои
- Данные приходится вводить 3 раза: блокнот → WhatsApp → 1С
- Нет актуальных остатков у торгпреда на точке

**Что ФерМама делает лучше прямо сейчас:**
- Живой расчёт суммы в реальном времени
- Красивый форматированный текст заявки
- Каталог с ценами и единицами измерения
- Отправка в один клик в нужный мессенджер
- Работает без интернета (статичный HTML)
- Нулевое внедрение, нулевая цена

## Типичные экраны в B2B приложениях для торговых представителей

### Обязательные экраны (MVP):

1. **Выбор менеджера / авторизация**
   - Список с именами (и аватарами в хорошем варианте)
   - PIN-код 4-6 цифр, не полный пароль — руки часто заняты
   - Запоминание последнего выбранного менеджера

2. **Список/поиск клиентов**
   - Живой поиск от 2-3 символов
   - История последних клиентов вверху
   - Возможность добавить нового вручную

3. **Карточка клиента**
   - История последних заказов + кнопка "Повторить"
   - Сумма задолженности (если актуально)
   - Контакты, адрес

4. **Каталог товаров**
   - Категории + поиск
   - Карточка: фото, название, цена, ед. изм., остаток
   - Stepper +/− прямо на карточке (без перехода)
   - Индикатор уже добавленных позиций

5. **Корзина / форма заявки**
   - Список позиций с редактированием количества
   - Живая сумма итого
   - Комментарий к заказу
   - Дата доставки

6. **Отправка / подтверждение**
   - Выбор мессенджера (Telegram / WhatsApp / Viber)
   - Превью текста перед отправкой
   - Чёткий статус: отправлено / ошибка

7. **История заявок** (важно, часто упускают)
   - Список с датой, клиентом, суммой
   - Кнопка "Повторить заказ" — одно из самых используемых действий

### Обязательные UI-элементы:

| Элемент | Зачем |
|---|---|
| Счётчик позиций + сумма | Всегда на виду, sticky footer |
| Stepper +/− | Минимум 44×44pt — пальцы в перчатках |
| Числовая клавиатура | При вводе количества — не QWERTY |
| Поиск по каталогу | 55 позиций — dropdown недостаточно |
| Поле "Клиент" с историей | Автодополнение / список последних |
| Поле "Комментарий" | Особые условия, пожелания |
| Кнопка отправки | Крупная, контрастная, в зоне большого пальца |
| Цена + единица измерения | шт / кг — рядом с товаром |
| Удаление позиции | Свайп или иконка корзины |
| Кнопка "Очистить" | С подтверждением |
| Бейдж "Акция" / "Нет в наличии" | Цветовая индикация |

### UX-паттерны которые работают лучше всего:

**Паттерн 1: "Быстрый старт"**
- Автоматически предлагать прошлый заказ клиента как шаблон
- Запоминать последнего клиента в сессии
- Кнопка "Повторить заказ" в истории — одно из самых используемых действий

**Паттерн 2: "Минимум тапов"**
- Добавление товара — 1 тап на +, без перехода на экран товара
- Изменение количества — прямо в корзине
- Создание заявки — не более 3 тапов от главного экрана

**Паттерн 3: "Офлайн-режим"**
- Магазины часто в зонах плохого сигнала (подвалы, склады)
- Каталог кэшируется локально
- Заявки сохраняются в черновик и отправляются при появлении сети

**Паттерн 4: "Одной рукой"**
- Торгпред держит телефон одной рукой
- Все ключевые действия — в нижней части экрана (зона большого пальца)
- Bottom navigation, не гамбургер-меню

**Паттерн 5: "Читаемость на улице"**
- Высокий контраст текста и фона
- Минимум 16px для основного текста
- Крупные кнопки с чёткими границами

**Паттерн 6: "Прощающий дизайн"**
- Корзина не очищается при случайном выходе
- Автосохранение черновика
- Возможность редактировать заявку до отправки
- Подтверждение только для деструктивных действий

### Типичные ошибки в плохих приложениях:

- Нет офлайн-режима — при потере сети всё зависает
- Маленькие кнопки +/− (< 36pt) — промахи при вводе
- QWERTY-клавиатура вместо цифровой при вводе количества
- Сброс корзины при сворачивании приложения
- Нет подтверждения успешной отправки
- Нет итоговой суммы во время набора (только в конце)
- Нет единицы измерения (штуки? ящики? кг?)
- Нет возможности изменить количество в корзине — только удалить и добавить заново

## Боли торговых представителей (реальные жалобы из отзывов)

| Боль | Как часто | Что из этого следует |
|------|-----------|----------------------|
| Прайс-лист устарел, цены неверные | Очень часто | Актуализация цен в приложении — критична |
| Не знают остатков на складе | Часто | Нужны живые остатки (или хотя бы пометка "нет") |
| Нет истории клиента под рукой | Часто | Карточка клиента с историей заказов |
| Двойной/тройной ввод данных | Большинство | Заявка уходит сразу, без перебивки |
| Нет фото товара на выезде | Почти всегда | Каталог с фото — сильное конкурентное преимущество |
| Приложение тормозит/сбоит | Часто (1С) | Простое HTML — никаких сбоев синхронизации |
| Клиент не видит подтверждения | Часто | Форматированное сообщение в мессенджер сразу |

> "Заявку принял в блокнот, потом набиваю в Excel, потом в 1С вводит оператор — тройное дублирование"
> "Обещал товар — его нет. Магазин злится. Я краснею."
> "Распечатал прайс в понедельник, в среду цены поменяли, стою перед клиентом с неверными ценами"

## Убийственные фичи которых нет у конкурентов

1. **"Покажи клиенту каталог с экрана"** — торгпред разворачивает телефон, клиент сам листает. Никто не сделал UX именно для этого сценария.
2. **Автоповтор заказа** — 70% заявок это стандартная корзина с небольшими изменениями. "Повторить как в прошлый раз" одной кнопкой.
3. **WhatsApp-подтверждение клиенту** — магазин получает сообщение с составом заказа сразу. Не нужно учить клиента новому сервису.
4. **Дебиторка перед визитом** — "долг 18 500 руб., срок 5 дней" — торгпред видит до входа в магазин.
5. **Order Matrix** — вместо 55 dropdown-строк — таблица: все товары сразу, вводишь числа там где надо.

## Что ФерМама может сделать лучше конкурентов

### Быстрые улучшения (текущий инструмент):
1. **Список клиентов** — автодополнение или история последних клиентов (localStorage)
2. **Быстрый ввод** — "+" / "−" кнопки рядом с количеством, не только текстовое поле
3. **Повторить заказ** — сохранить последний заказ клиента и загрузить одной кнопкой
4. **Поиск по каталогу** — поле поиска при добавлении товара (сейчас dropdown из 55 позиций)
5. **История** — локальное хранение последних 10 заявок в localStorage
6. **Order Matrix (таблица)** — радикальная альтернатива текущим dropdown-строкам: каталог в виде таблицы, поле количества рядом с каждым товаром. Торгпред проходит по списку сверху вниз, вводит нужные числа, пропускает ненужные. Это в 3-5 раз быстрее текущего подхода.

### Стратегические преимущества перед крупными системами:
1. **Нулевое внедрение** — открыл, начал работать. Никакой настройки
2. **Нулевая цена** — статичный файл, нет подписки
3. **Мессенджеры** — Telegram/WhatsApp — родная среда для российского бизнеса
4. **Простота** — 3 шага: кто ты → кто клиент → что берёт → отправить
5. **Форматированный текст** — красивое сообщение сразу готово к отправке
6. **Работает на любом телефоне** — не нужно скачивать приложение
